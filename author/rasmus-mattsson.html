Global Competence Hub, Ericsson AB

Title: Integration Engineer
Period: 2013 - Now

Description

In the Global Competence Hub, Rasmus worked in a large project to automate fault management of network nodes in Ericssons managed services programme. The automation was implemented by providing a framework in which repair procedures could be easily be constructed in a graphical programming language. The repair procedures would then automatically run on specific error conditions in network nodes.

Example of Accomplishments

  • Implementation of a proxy mechanism in CORBA that could route traffic from an Ericsson managed network through a demilitarized zone into a managed operator network and back. This mechanism enables communication with network nodes.
  • Packaging of components for installation with RPM, the Red Hat Linux package manager.
  • Implementation of a Continuous Integration framework for the solution in Python.

Domain Competences

Backend Programming

Most of his professional career Rasmus has spent writing backends that runs on various Unix and Linux platforms. He has good knowledge of both Java and Python commonly used in this domain as well as agile methodologies and test driven development (TDD).

Mobile Application Development

Rasmus has been working professionally with Mobile Application Development for about three years, developing a VoIP and chat client for three different platforms: Android, Blackberry and Symbian. Although two out of these three platforms are (almost) not used today they provided Rasmus with a solid understanding of mechanisms that gives him an easy time to pick up new technologies related to mobile application development at any time.

Web Development

Over the years Rasmus has (mostly out of curiosity) developed a solid understanding fundamental web technologies such as HTTP, HTML5, CSS and Javascript as well as common web utilities like jQuery and Bootstrap. Rasmus is also good at handling GIMP, an advanced image manipulation program and can to a degree create and modify his own graphical elements.

Title: Mobile Application Developer
Period: 2010 - 2012

Description

Rasmus learnt how to combine security requirements with user friendliness and how to handle availability and VoIP in shaky mobile networks with low throughput.
